Types do Formato

Cada type define o contrato de valor de $value. Aqui você vê a regra e um exemplo de aplicação real lado a lado.

Tópicos

  • color
  • dimension
  • fontFamily
  • fontWeight
  • duration
  • cubicBezier
  • number
  • additional types

Leitura rápida: se o type está correto, ferramentas conseguem validar, transformar e distribuir tokens com menos ambiguidade.

color

UI Surface

Representa cores semânticas da interface, como texto, fundo e ações.

  • Use nomes semânticos, não nomes de cor literal.
  • Evita retrabalho quando a paleta visual muda.
{
  "color": {
    "text": { "primary": { "$type": "color", "$value": "#14213d" } }
  }
}

.title { color: var(--color-text-primary); }

dimension

Size + Unit

Define medidas com unidade explícita px ou rem.

  • Ideal para espaçamentos, raio e tamanhos.
  • O valor precisa ser objeto com value e unit. 
{
  "spacing": {
    "md": { "$type": "dimension", "$value": { "value": 16, "unit": "px" } }
  }
}

.card { padding: var(--spacing-md); }

fontFamily

Typography

Aceita string única ou lista ordenada de fallback.

  • Preferir lista quando o design depende de fallback controlado.
  • Evite misturar role de heading/body no mesmo token.
{
  "font": {
    "body": { "$type": "fontFamily", "$value": ["Space Grotesk", "Arial", "sans-serif"] }
  }
}

body { font-family: var(--font-body); }

fontWeight

1..1000

Peso tipográfico como inteiro de 1 a 1000 ou alias como bold.

  • Numérico facilita mapeamento entre plataformas.
  • Alias pode ajudar legibilidade em bibliotecas de token.
{
  "font": {
    "heading-weight": { "$type": "fontWeight", "$value": 700 }
  }
}

h1, h2 { font-weight: var(--font-heading-weight); }

duration

Motion Time

Representa tempo de transição/animação com value e unidade ms ou s.

  • Mantenha escala curta, média e longa para consistência.
  • Evite valores arbitrários fora da escala definida.
{
  "motion": {
    "fast": { "$type": "duration", "$value": { "value": 180, "unit": "ms" } }
  }
}

button { transition-duration: var(--motion-fast); }

cubicBezier

Easing Curve

Curva de easing com quatro números: [P1x, P1y, P2x, P2y].

  • P1x e P2x precisam ficar entre 0 e 1.
  • Útil para criar assinatura de movimento do produto.
{
  "motion": {
    "entrance-ease": { "$type": "cubicBezier", "$value": [0.2, 0.8, 0.2, 1] }
  }
}

.modal { transition-timing-function: var(--motion-entrance-ease); }

number

Unitless

Número puro sem unidade, ideal para relações proporcionais.

  • Use para line-height, opacidade e escalas matemáticas.
  • Não inclua sufixo de unidade dentro do valor.
{
  "text": {
    "line-height-body": { "$type": "number", "$value": 1.5 }
  }
}

p { line-height: var(--text-line-height-body); }

additional types

Não normativo

A especificação cita possíveis evoluções, mas sem exigência normativa atual.

  • Use apenas se seu pipeline aceitar extensões customizadas.
  • Documente bem para facilitar migração futura.
{
  "font": {
    "body-style": { "$type": "fontStyle", "$value": "italic" }
  }
}

.quote { font-style: var(--font-body-style); }
