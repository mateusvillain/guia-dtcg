Formatação Geral de Design Token

Uma leitura progressiva da estrutura oficial de token: o que é obrigatório, o que é metadado e o que precisa de atenção na sintaxe.

Tópicos

  • Estrutura e valor
  • Nome e sintaxe
  • Metadados reservados
  • Exemplo completo

Resumo: todo token é um par nome + objeto, e esse objeto precisa ter $value. O restante adiciona semântica e governança.

1. Estrutura e valor

1

Nome do token

É a chave no objeto JSON. Exemplo: "color-text-primary".

2

Corpo do token

É um objeto com propriedades reservadas do padrão. A única obrigatória sempre é $value. 

{
  "color-text-primary": {
    "$value": "#111111"
  }
}

2. Nome e sintaxe

Case-sensitive

Color.Primary e color.primary são nomes diferentes em JSON.

Sem prefixo $

Tokens e grupos não devem começar com $.

Sem { } ou .

Esses caracteres são reservados pela sintaxe do ecossistema de tokens.

3. Metadados reservados

1

$description

Descrição textual opcional do propósito do token.

2

$extensions

Bloco opcional para dados proprietários. Use namespace claro, como domínio reverso.

3

$deprecated

Pode ser true, false ou uma string explicando substituição/migração.

4. Exemplo completo

{
  "color-text-primary": {
    "$type": "color",
    "$description": "Cor principal para leitura em superfícies padrão.",
    "$value": "#14213d",
    "$extensions": {
      "com.exemplo.docs": {
        "figmaStyleId": "S:1234"
      }
    },
    "$deprecated": false
  }
}
