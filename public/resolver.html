Resolver: conceito e criação

Como combinar sets e modifiers por contexto usando resolutionOrder.

O que o resolver faz

  • Entrada: parâmetros de contexto (ex.: theme=dark, contrast=high).
  • Processo: valida parâmetros, aplica fontes conforme a ordem, resolve aliases.
  • Saída: conjunto final de tokens para a combinação de contextos.

Estrutura básica

{
  "$schema": "https://www.designtokens.org/schemas/2025.10/resolver.json",
  "version": "2025.10",
  "sets": {
    "base": { "sources": [{ "color": { "text": { "primary": { "$type": "color", "$value": "#111111" } } } }] }
  },
  "modifiers": {
    "theme": {
      "contexts": {
        "light": [{ "color": { "background": { "$type": "color", "$value": "#FFFFFF" } } }],
        "dark": [{ "color": { "background": { "$type": "color", "$value": "#121212" } } }]
      },
      "default": "light"
    }
  },
  "resolutionOrder": [
    { "$ref": "#/sets/base" },
    { "$ref": "#/modifiers/theme" }
  ]
}

Como criar

  1. Comece por um set base com tokens comuns.
  2. Adicione modifiers para variáveis de contexto (tema, contraste, densidade etc.).
  3. Defina contexto padrão em cada modifier.
  4. Ordene composição em resolutionOrder do mais base ao mais específico.
  5. Teste combinações de contextos para validar conflitos e overrides.
